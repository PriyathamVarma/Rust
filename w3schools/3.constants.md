# 🦀 Rust Constants

## 1. 📌 What are Constants?

Constants are variables whose **values never change** during program execution.

💡 **Key Idea:**  
Once defined → value is fixed forever ❌🔁

---

## 2. 🛠️ Creating a Constant

Use the `const` keyword along with:
- Name  
- Type  
- Value  

### Example:

```rust
fn main() {
    const BIRTHYEAR: i32 = 1980;
    const MINUTES_PER_HOUR: i32 = 60;

    println!("Birth Year: {}", BIRTHYEAR);
    println!("Minutes in an hour: {}", MINUTES_PER_HOUR);
}
````

---

## 3. ⚠️ Constants Must Have a Type

Unlike variables, Rust **does NOT infer types for constants**.

### ✅ Correct:

```rust
const BIRTHYEAR: i32 = 1980;
```

### ❌ Wrong:

```rust
const BIRTHYEAR = 1980; // Error: type missing
```

💡 **Rule:**
Always specify type when using `const`.

---

## 4. 🔤 Naming Rules (Best Practice)

Constants are usually written in **UPPERCASE**.

### Examples:

* `MAX_SPEED`
* `PI`
* `MINUTES_PER_HOUR`

💡 This improves:

* Readability 👀
* Code consistency

---

## 5. ⚖️ Constants vs Variables

| Feature        | Constant (`const`) ❌🔁 | Variable (`let`) 🔁 |
| -------------- | ---------------------- | ------------------- |
| Can change?    | ❌ No                   | ✅ Yes (with `mut`)  |
| Type required? | ✅ Yes                  | ❌ Optional          |
| Keyword used   | `const`                | `let`               |

---

## 🧠 Quick Revision

* `const` → used for fixed values
* Must always define **type** ⚠️
* Value cannot change ❌
* Naming usually in **UPPERCASE**
* Different from `let` variables
