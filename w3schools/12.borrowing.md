# 🦀 Rust Borrowing and References

## 1. 📌 What is Borrowing?

Borrowing means **using a value without taking ownership of it**.

💡 Instead of moving or copying data, Rust allows you to **borrow it safely**.

---

## 2. 🔗 What is a Reference?

A reference allows you to **access a value without owning it**.

- Created using `&`
- Does NOT take ownership

---

### Example:

```rust
fn main() {
    let a = String::from("Hello");
    let b = &a;

    println!("a = {}", a);
    println!("b = {}", b);
}
````

### 🔍 Explanation:

* `a` → owner of the value
* `b` → reference (borrower)
* Both can be used safely ✅

💡 Ownership stays with `a`

---

## 3. ✏️ Mutable References

Used when you want to **modify a value through a reference**.

* Created using `&mut`

---

### Example:

```rust
fn main() {
    let mut name = String::from("John");

    let name_ref = &mut name;
    name_ref.push_str(" Doe");

    println!("{}", name_ref);
}
```

### Output:

```
John Doe
```

---

## 4. ⚠️ Important Rule (Very Important)

You can only have:

* ✅ **One mutable reference at a time**
* ❌ Cannot have multiple mutable references simultaneously

💡 This prevents:

* Data conflicts
* Unexpected bugs

---

## 5. 🔄 Borrowing vs Ownership

| Concept   | Behavior                  |
| --------- | ------------------------- |
| Ownership | Transfers value           |
| Borrowing | Uses value without owning |

---

## 6. 🎯 Why Borrowing is Important

Borrowing helps:

* 🔁 Reuse values without losing ownership
* 🚫 Avoid unnecessary cloning (better performance)
* 🛡️ Prevent memory errors
* ⚡ Improve efficiency

---

## 🧠 Quick Revision

* `&` → reference (borrow value)
* `&mut` → mutable reference ✏️
* Borrowing does NOT transfer ownership
* Only one mutable reference allowed ⚠️
* Makes Rust safe and efficient 🛡️
