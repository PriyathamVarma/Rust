# 🦀 Rust Ownership

## 1. 📌 What is Ownership?

Ownership is Rust’s system for **managing memory safely**.

💡 Instead of using a garbage collector, Rust uses **ownership rules** to:
- Prevent memory errors 🚫  
- Automatically clean up memory 🧹  
- Keep programs fast ⚡  

---

## 2. 📜 Ownership Rules

1. Each value has **one owner**  
2. When the owner goes out of scope → value is **deleted**  
3. Only one owner at a time (unless borrowed)  

---

## 3. 🔁 Basic Ownership Example (Move)

```rust
fn main() {
    let a = String::from("Hello");
    let b = a;

    // println!("{}", a); ❌ Error
    println!("{}", b); // ✅ Works
}
````

### 🔍 Explanation:

* `a` owns the string initially
* `b = a` → ownership is **moved** to `b`
* `a` becomes invalid ❌

💡 This is called a **move**

---

## 4. 📋 Copy vs Move

### 🔹 Simple Types (Copy)

Numbers, characters, and booleans are **copied**, not moved.

```rust id="2h6m4w"
fn main() {
    let a = 5;
    let b = a;

    println!("a = {}", a);
    println!("b = {}", b);
}
```

### 🔍 Explanation:

* `a` is copied into `b`
* Both variables work independently

---

### 🔹 Complex Types (Move)

Types like `String` are **moved**, not copied.

```rust id="x0kq3y"
let a = String::from("Hello");
let b = a; // move happens here
```

---

## 5. 📑 Clone (Deep Copy)

If you want **two separate owners**, use `.clone()`.

```rust id="1gl1y6"
fn main() {
    let a = String::from("Hello");
    let b = a.clone();

    println!("a = {}", a);
    println!("b = {}", b);
}
```

### 🔍 Explanation:

* `.clone()` creates a **full copy of data**
* Both `a` and `b` are valid

💡 Use clone carefully (it uses more memory)

---

## 6. ⚠️ Important Note

Instead of cloning, Rust often prefers **borrowing (`&`)**
➡️ Covered in next topic

---

## 7. 🎯 Why Ownership Matters

Ownership helps Rust:

* 🧹 Automatically free memory
* 🚫 Prevent invalid memory access
* ⚡ Maintain high performance
* 🛡️ Ensure safety without garbage collection

---

## 🧠 Quick Revision

* Every value has **one owner**
* Assigning complex types → **move** 🔁
* Simple types → **copy** 📋
* Use `.clone()` for deep copy 📑
* Ownership prevents memory bugs 🛡️
